# Talent readiness

Odd One is designed for genuine play rather than transaction generation.

For Celo, the implementation provides MiniPay detection, automatic wallet access, Viem transactions, mainnet-ready deployment, source verification, and a mobile layout. Proof of Ship still requires an actual mainnet deployment and project submission.

For Stacks, the implementation uses Clarity, `@stacks/connect`, and `@stacks/transactions`. Builder evaluation still requires a public GitHub repository and real deployed activity.

Automated activity, multiple accounts used to inflate metrics, bots, and leaderboard manipulation are outside the product and prohibited. QA simulations are restricted to local networks and recognized testnets.
